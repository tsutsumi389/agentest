# セキュリティポリシー

## 脆弱性の報告

セキュリティに関する問題を発見した場合は、**公開 Issue を作成せず**、以下の方法で報告してください。

### GitHub Security Advisories（推奨）

[GitHub Security Advisories](https://github.com/tsutsumi389/agentest/security/advisories/new) を使用して非公開で報告してください。

### 報告に含める情報

- 脆弱性の説明
- 再現手順
- 影響範囲の評価
- 可能であれば修正案

### 対応フロー

1. 報告受領後、3営業日以内に確認の連絡をします
2. 脆弱性の評価・修正を行います
3. 修正版をリリースします
4. 報告者にクレジットを付与します（希望する場合）

## セルフホスト環境のセキュリティ推奨事項

Agentest をセルフホストする際は、以下のセキュリティ対策を推奨します。

### 必須

- **HTTPS の有効化**: リバースプロキシ（nginx, Caddy 等）で TLS を設定
- **強力なシークレット**: `JWT_SECRET`、`TOKEN_ENCRYPTION_KEY`、`TOTP_ENCRYPTION_KEY` に十分にランダムな値を設定
- **メール認証**: 本番環境では `REQUIRE_EMAIL_VERIFICATION=true`（デフォルト）を使用

### 推奨

- **ファイアウォール**: データベース・Redis ポートへの外部アクセスを制限
- **定期的な更新**: Docker イメージとパッケージを最新に保つ
- **バックアップ**: データベースの定期バックアップを設定
- **ログ監視**: 異常なアクセスパターンを監視
- **管理者 2FA**: 管理者アカウントに TOTP 二要素認証を有効化

### 環境変数の管理

- シークレットを Git リポジトリにコミットしない
- 環境変数ファイル（`.env`）のパーミッションを制限（`chmod 600`）
- 本番環境ではシークレット管理ツールの使用を検討

## サポートバージョン

最新のメジャーバージョンのみセキュリティ修正を提供します。
