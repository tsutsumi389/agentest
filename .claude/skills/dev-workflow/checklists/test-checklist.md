# Test Checklist

テスト完了前の確認事項。

## カバレッジ

- [ ] Line Coverage >= 80%
- [ ] Branch Coverage >= 75%
- [ ] Function Coverage >= 80%
- [ ] 重要なビジネスロジックは90%以上

## ユニットテスト

- [ ] Service層のテストが作成されている
- [ ] Repository層のテストが作成されている
- [ ] 正常系がテストされている
- [ ] 異常系（エラーケース）がテストされている
- [ ] 境界値がテストされている
- [ ] モックが適切に使用されている

## 結合テスト

- [ ] 主要APIエンドポイントがテストされている
- [ ] 正常系レスポンスが検証されている
- [ ] エラーレスポンス（400, 401, 403, 404, 409等）が検証されている
- [ ] リクエストバリデーションがテストされている
- [ ] 認証・認可がテストされている

## テスト品質

- [ ] テストが独立して実行可能
- [ ] テストが再現可能（冪等）
- [ ] テスト名が何をテストしているか明確
- [ ] アサーションが具体的で明確
- [ ] テストデータが適切

## CI/CD

- [ ] 全テストがCIで実行可能
- [ ] テスト実行時間が妥当（遅すぎない）

## ドキュメント

- [ ] テスト実行方法がREADMEに記載されている
- [ ] 特殊なテスト設定があれば記載されている
