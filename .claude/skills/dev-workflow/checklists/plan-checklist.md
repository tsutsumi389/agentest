# Plan Checklist

実装プラン完了前の確認事項。

## 要件

- [ ] 機能要件が明確に定義されている
- [ ] 非機能要件（パフォーマンス、セキュリティ等）が考慮されている
- [ ] ユーザーストーリーまたはユースケースが記載されている
- [ ] 受け入れ条件が明確

## 技術設計

- [ ] API設計（エンドポイント、リクエスト/レスポンス形式）
- [ ] データモデル設計（新規/変更テーブル）
- [ ] 主要コンポーネントと責務が定義されている
- [ ] 既存コードへの影響が特定されている

## タスク分解

- [ ] 実装可能な粒度に分解されている
- [ ] 各タスクに見積もりがある
- [ ] 依存関係が明確
- [ ] 優先順位が設定されている

## リスク管理

- [ ] 技術的リスクが洗い出されている
- [ ] リスクへの対策が記載されている
- [ ] 未決定事項がリストアップされている

## その他

- [ ] 関連Issueへのリンクがある
- [ ] テスト方針が記載されている
- [ ] ドキュメント更新範囲が特定されている
