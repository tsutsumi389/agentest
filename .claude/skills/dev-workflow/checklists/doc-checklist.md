# Documentation Checklist

ドキュメント完了前の確認事項。

## API仕様書

- [ ] 全ての新規エンドポイントが記載されている
- [ ] リクエスト形式（Headers, Body）が正確
- [ ] レスポンス形式（Success, Error）が正確
- [ ] 必須/任意パラメータが明記されている
- [ ] サンプルリクエスト/レスポンスがある
- [ ] エラーコードと説明がある

## README

- [ ] 新機能がFeaturesセクションに追加されている
- [ ] セットアップ手順が更新されている（必要な場合）
- [ ] 依存関係が更新されている（必要な場合）

## 環境変数

- [ ] 新規環境変数が.env.exampleに追加されている
- [ ] 環境変数の説明が記載されている
- [ ] Required/Optionalが明記されている
- [ ] デフォルト値がある場合は記載されている

## 技術ドキュメント（該当する場合）

- [ ] アーキテクチャ/設計が説明されている
- [ ] データフローが説明されている
- [ ] 設定方法が説明されている
- [ ] トラブルシューティングがある

## 品質

- [ ] 誤字脱字がない
- [ ] 用語が一貫している
- [ ] フォーマットが既存ドキュメントと統一されている
- [ ] リンクが有効
- [ ] サンプルコードが動作する

## レビュー

- [ ] 技術的な正確性が確認されている
- [ ] 初見で理解できる内容になっている
