# Security Checklist

セキュリティレビュー時の確認事項。

## 入力検証

- [ ] 全ての外部入力がバリデーションされている
- [ ] SQLインジェクション対策（パラメータ化クエリ、Prisma使用）
- [ ] XSS対策（HTMLエスケープ、サニタイズ）
- [ ] パストラバーサル対策（ファイルパス検証）
- [ ] コマンドインジェクション対策（シェル実行の回避）
- [ ] 正規表現DoS（ReDoS）対策
- [ ] JSONパース時のプロトタイプ汚染対策

## 認証・認可

- [ ] 認証が必要なエンドポイントに認証ミドルウェアがある
- [ ] JWT検証が適切（署名、有効期限、issuer）
- [ ] リソースへのアクセス権限チェックがある
- [ ] 権限昇格の脆弱性がない（IDOR対策）
- [ ] ブルートフォース対策（ログイン試行制限）
- [ ] パスワードリセットフローが安全

## パスワード・シークレット

- [ ] パスワードは適切にハッシュ化（bcrypt, argon2）
- [ ] ソルトが使用されている
- [ ] APIキー・シークレットがコードにハードコードされていない
- [ ] .envファイルが.gitignoreに含まれている
- [ ] 本番シークレットが開発環境に露出していない

## データ保護

- [ ] 個人情報（PII）の取り扱いが適切
- [ ] センシティブデータの暗号化（保存時・転送時）
- [ ] 不要なデータの保持を避けている
- [ ] データ削除時の完全消去

## API セキュリティ

- [ ] レートリミットが設定されている
- [ ] CORS設定が適切（許可オリジンの制限）
- [ ] HTTPSが強制されている（本番環境）
- [ ] セキュリティヘッダーが設定されている
  - [ ] `X-Content-Type-Options: nosniff`
  - [ ] `X-Frame-Options: DENY`
  - [ ] `Strict-Transport-Security`
  - [ ] `Content-Security-Policy`
- [ ] 不要なHTTPメソッドが無効化されている

## エラーハンドリング

- [ ] スタックトレースが本番環境で露出しない
- [ ] エラーメッセージが内部情報を漏らさない
- [ ] 存在確認エラー（ユーザー存在の有無など）が統一されている

## ログ・監査

- [ ] パスワード、トークンがログに出力されない
- [ ] クレジットカード番号等がマスクされている
- [ ] 認証失敗、権限エラーがログに記録される
- [ ] 重要な操作が監査ログに記録される

## 依存関係

- [ ] `npm audit` で脆弱性がない（または対応済み）
- [ ] 依存パッケージが最新に近い
- [ ] 不要な依存関係が削除されている
- [ ] lockファイルがコミットされている

## セッション管理

- [ ] セッションIDが十分にランダム
- [ ] セッションの有効期限が適切
- [ ] ログアウト時にセッションが無効化される
- [ ] セッション固定攻撃対策

## ファイルアップロード（該当する場合）

- [ ] ファイルタイプの検証（拡張子だけでなくMIMEタイプも）
- [ ] ファイルサイズの制限
- [ ] アップロード先がWebルート外
- [ ] ファイル名のサニタイズ
- [ ] マルウェアスキャン（必要に応じて）

## 設定・デプロイ

- [ ] デバッグモードが本番で無効
- [ ] 不要なエンドポイント（/debug等）が本番で無効
- [ ] 環境変数が正しく設定されている
- [ ] デフォルトの認証情報が変更されている

## 定期チェック

```bash
# 依存関係の脆弱性チェック
npm audit

# 修正可能な脆弱性を自動修正
npm audit fix

# 詳細レポート
npm audit --json
```
