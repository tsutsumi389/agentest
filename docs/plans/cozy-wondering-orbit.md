# テスト実施結果の概要レイアウト変更計画

## 概要
テスト実施結果の概要（ExecutionOverviewPanel）のレイアウトを変更し、期待結果の成功/失敗を見やすくする。

## 変更対象
- `/apps/web/src/components/execution/ExecutionOverviewPanel.tsx`

## 現在のレイアウト順序
1. ヘッダー（テストスイート名、ステータス）
2. メタデータカード（開始日時/終了日時/環境）
3. **前提条件サマリー**（満たす/満たさない/未確認）
4. **手順サマリー**（完了/スキップ/未実行）
5. **期待結果サマリー**（成功/失敗/スキップ/実行不可/未実行）
6. スイートレベル前提条件リスト

## 変更後のレイアウト順序
1. ヘッダー
2. メタデータカード
3. **期待結果サマリー**（成功/失敗/スキップ/実行不可/未実行）← 一番上に移動
4. **前提条件サマリー**（満たす/満たさない/未確認）
5. **手順サマリー**（完了/スキップ/未実行）
6. スイートレベル前提条件リスト

## 実装内容

### 1. セクション順序の変更（必須）
期待結果サマリーを前提条件サマリーの前に移動（行356-394を行296の前に移動）

### 2. 期待結果セクションの強調表示
成功/失敗を視覚的に目立たせるため、以下を追加：

```
┌──────────────────────────────────────────────────────────────┐
│ 期待結果                                    成功率: 80%       │
│ ████████████████████░░░░░░  (12/15 実行済み)                │
│                                                              │
│ ┌──────────────┐ ┌──────────────┐ ┌─────┐ ┌─────┐ ┌─────┐   │
│ │ ✓ 12 成功   │ │ ✗ 3 失敗    │ │スキ │ │実行 │ │未実 │   │
│ │ (強調表示)  │ │ (強調表示)  │ │ップ │ │不可 │ │ 行  │   │
│ └──────────────┘ └──────────────┘ └─────┘ └─────┘ └─────┘   │
└──────────────────────────────────────────────────────────────┘
```

- プログレスバーを追加（既存の`ProgressBar`コンポーネントを拡張）
- 成功/失敗カードは背景色＋ボーダーで強調（`bg-success-subtle border-success/30`）
- 成功率をパーセンテージで表示

### 3. 前提条件・手順セクションはそのまま
既存の3カラムレイアウトを維持

## 実装ステップ

1. **ProgressBar拡張**
   - `notExecutable`プロパティを追加（アクセントカラーで表示）

2. **ExpectedResultsHighlightSummary コンポーネント作成**
   - ヘッダー行（ラベル + 成功率）
   - プログレスバー
   - 成功/失敗を強調したカードグリッド

3. **ExecutionOverviewPanel修正**
   - セクション順序を変更
   - 期待結果セクションを新コンポーネントに置き換え

## 検証方法
1. `docker compose exec dev pnpm build` でビルドエラーがないことを確認
2. ブラウザでテスト実行画面を表示し、以下を確認：
   - 期待結果が最上部に表示される
   - 成功/失敗が視覚的に目立つ
   - プログレスバーが正しく表示される
   - レスポンシブ対応（モバイル表示）
