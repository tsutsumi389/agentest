# 実行結果概要画面の改善プラン

## 概要
ExecutionOverviewPanelの視認性向上と情報の充実化

## 変更内容

### 1. メタデータ表示の改善（カード3列）
現在の一行テキスト表示から、3つのカードに分離：

| カード | アイコン | 内容 |
|--------|---------|------|
| 開始日時 | Calendar | 実行開始日時 |
| 終了/経過 | Timer | 終了日時 + 経過時間（実行中は「実行中...」） |
| 実行環境 | Server | 環境名（未設定時は「未設定」） |

### 2. サマリーの3段構成化
現在の期待結果のみから、3種類のサマリーに拡張：

**前提条件サマリー（3カード）** - データがある場合のみ表示
- 満たす (MET) - 緑
- 満たさない (NOT_MET) - 赤
- 未確認 (UNCHECKED) - グレー

**手順サマリー（3カード）** - データがある場合のみ表示
- 完了 (DONE) - 緑
- スキップ (SKIPPED) - 黄
- 未実行 (PENDING) - グレー

**期待結果サマリー（5カード）**
- 成功 (PASS) - 緑
- 失敗 (FAIL) - 赤
- スキップ (SKIPPED) - 黄
- 実行不可 (NOT_EXECUTABLE) - 紫
- 未実行 (PENDING) - グレー

## 変更ファイル

### 1. `apps/web/src/lib/date.ts`
- `formatDuration(start, end)` 関数を追加
- 経過時間を「1時間23分45秒」形式でフォーマット

### 2. `apps/web/src/components/execution/ExecutionOverviewPanel.tsx`
- `MetadataCard` コンポーネントを追加
- サマリー計算を3種類に拡張（preconditionSummary, stepSummary, expectedSummary）
- 期待結果サマリーで SKIPPED と NOT_EXECUTABLE を分離（5カードに）
- ヘッダー部のテキストを削除、メタデータカードセクションを追加
- サマリーセクションを3段構成に変更
- ガイドテキスト（「サイドバーからテストケースを選択して...」）を削除
- 必要なアイコンをインポート追加（Calendar, Timer, Server, Circle, MinusCircle, ListChecks, ClipboardCheck, Slash）

## 検証方法
1. `docker compose exec dev pnpm build` でビルドエラーがないことを確認
2. ブラウザで実行結果画面を開き、以下を確認：
   - メタデータカード3列が正しく表示される
   - 各サマリーセクションの件数が正しい（期待結果は5カード）
   - レスポンシブ（モバイル時は縦並び）が機能する
   - 前提条件/手順がない場合、該当セクションが非表示になる
   - ガイドテキストが削除されている
