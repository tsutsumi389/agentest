# Agentest 実装順序計画

## 概要

`docs/requirements/` 配下の要件ドキュメント（test-management.md、user-organization.md）の実装順序を決定する。

## 依存関係の分析

機能間の依存関係を分析した結果：

- **ユーザー管理** → 全ての機能の基盤
- **組織管理** → ユーザー管理に依存
- **メンバー管理** → 組織管理に依存
- **プロジェクト管理** → ユーザー/組織管理に依存（プロジェクトは個人または組織に所属）
- **テストスイート管理** → プロジェクト管理に依存
- **テストケース管理** → テストスイート管理に依存
- **テスト実行** → テストケース管理に依存
- **Agent連携** → テスト管理機能全体に依存
- **レビュー/同時編集** → テストスイート/テストケースに依存
- **課金/通知/監査** → 基本機能が揃った後

## 実装順序の根拠

1. **Phase 1-2**: 認証と組織管理が全機能の土台
2. **Phase 3-5**: プロジェクト → スイート → ケースの階層構造に従う
3. **Phase 6**: テストケースが存在して初めて実行可能
4. **Phase 7**: レビューはテスト管理の付加価値機能
5. **Phase 8-9**: Agent連携は人による操作が確立した後
6. **Phase 10-13**: 運用機能は基本機能完成後
7. **Phase 14**: 補完的な機能は最後
