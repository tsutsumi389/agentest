# PiPレイアウト変更プラン

## 概要
テスト実施のPicture-in-Pictureパネルのレイアウトを改善する。

## 対象ファイル
- `apps/web/src/components/execution/PipExecutionPanel.tsx`

## 変更要件

### 1. 縦の余白を削除
**現状**: `min-h-full` で下に余白が残る
**変更**: `h-full` に変更し、ウィンドウいっぱいに表示

### 2. ステータスとノートの順序入れ替え
**現状**: コンテンツ → ステータス → ノート
**変更**: コンテンツ → ノート → ステータス

### 3. ステータスボタンのサイズを大きく
**現状**: `px-2 py-1 text-xs` + `w-3 h-3`アイコン
**変更**: `px-3 py-2 text-sm` + `w-4 h-4`アイコン（縦を約2倍に）

### 4. ステータスクリックで次のアイテムへ自動遷移
**現状**: ステータス変更のみ
**変更**: ステータス変更後に自動で次のアイテムへ移動
- どのステータスを選んでも次のアイテムへ遷移する
- 最後のアイテムの場合は次のテストケースへ移動する
- `PipStatusButtons`コンポーネントに`onNavigateNext`プロップを追加
- ステータス変更のコールバック内で次へ遷移を実行

## 実装手順

1. `PipExecutionPanel`のルート`div`を`min-h-full`→`h-full`に変更
2. メインコンテンツ内のステータスセクションとノートセクションの順序を入れ替え
3. `PipStatusButtons`のボタンスタイルを更新（サイズを大きく）
4. `PipStatusButtons`に`onNavigateNext`コールバックを追加し、クリック時に呼び出す

## 検証方法
1. PiPウィンドウを開く
2. 下に余白がないことを確認
3. ノートがステータスの上に表示されることを確認
4. ステータスボタンが大きくなっていることを確認
5. ステータスをクリックして次のアイテムに自動で移動することを確認
6. 最後のアイテムでステータスをクリックして次のテストケースへ移動することを確認
