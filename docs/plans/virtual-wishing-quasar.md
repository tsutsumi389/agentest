# テストスイート一覧 フィルター機能改善計画

## 概要
テストスイート一覧画面のフィルター機能を改善し、使いやすさを向上させる。

## 変更内容

### 1. 削除する機能
- 作成者フィルター（`createdBy`）
- 作成日フィルター（`from` / `to`）

### 2. 追加する機能
- **ラベルフィルター**: 複数選択可能、**OR条件**（いずれかのラベルにマッチすれば表示）、既存の `LabelSelector` コンポーネントを活用

### 3. デフォルト値の変更
| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| ステータス | すべて | `ACTIVE`（有効） |
| ソート | `createdAt`（作成日時）降順 | `updatedAt`（更新日時）降順 |

### 4. UI/UX改善
現在のドロップダウンパネル形式から、**インラインフィルター形式**に変更。

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [🔍 テストスイートを検索...]  [ステータス: 有効 ▼]  [ラベル ▼]          │
│                                                                         │
│  並び順: [更新日時 ▼] [降順 ▼]   □ 削除済みも表示       10件のテストスイート│
├─────────────────────────────────────────────────────────────────────────┤
│  選択中: [バグ修正 ✕] [UI ✕]                          [すべてクリア]     │
└─────────────────────────────────────────────────────────────────────────┘
```

**改善ポイント**:
- フィルター状態が常に見える（ドロップダウンを開かなくても確認可能）
- 操作がシンプル（1クリックでフィルター変更）
- 選択中のラベルがバッジで視覚的に表示される

## 検証方法
1. `docker compose up` で開発サーバー起動
2. テストスイート一覧画面を開く
3. 以下を確認:
   - デフォルトで「有効」ステータスのみ表示されること
   - デフォルトで更新日時の降順でソートされていること
   - ラベルフィルターで複数選択できること
   - 作成者・作成日フィルターが削除されていること
   - 選択中のフィルターがバッジで表示されること
