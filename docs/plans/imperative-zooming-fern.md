# 最近の活動セクションの高さ調整

## 目的
プロジェクト概要タブの「最近の活動」セクションの高さを5つの活動アイテムが表示される高さに固定し、それ以上のアイテムはスクロールで表示する。

## 現状の問題
- 「最近の活動」が全てのアイテムを表示するため、左側の「実行結果の分布」チャートと高さが合わない
- 見た目のバランスが悪い

## 変更対象ファイル
- `apps/web/src/components/project/dashboard/RecentActivityTimeline.tsx`

## 実装内容

### アクティビティリストに最大高さを設定

各`ActivityItem`は以下の構成：
- `p-3` (padding 12px)
- アイコン: `h-8` (32px)
- テキスト + アクター情報

推定高さ：約68px/アイテム
5アイテム + 4ギャップ(8px) = 5 × 68 + 4 × 8 = 340 + 32 = 372px

`max-h-[372px]` または類似の値をアクティビティリストに適用

**変更箇所** (79行目付近):
```tsx
// 変更前
<div className="flex-1 overflow-y-auto space-y-2">

// 変更後
<div className="overflow-y-auto space-y-2 max-h-[372px]">
```

## 検証方法
1. 開発サーバーを起動
2. プロジェクト概要タブを開く
3. 「最近の活動」セクションが5アイテム分の高さになっていることを確認
4. 5アイテム以上ある場合、スクロールで他のアイテムが表示されることを確認
