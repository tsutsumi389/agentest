# 編集・作成モード時のヘッダー非表示対応

## 概要
テストスイートの編集中、およびテストケースの編集・作成中に、タイトルとタブが表示されるヘッダー部分を非表示にする。

## 現状の問題
編集・作成モードでもヘッダー（タイトル「入札者ログイン機能」とタブ「概要」「実行履歴」「レビュー」「変更履歴」「設定」）が表示されている。

### 現在の画面構成
```
┌─────────────────────────────────────────┐
│ タイトル: 「入札者ログイン機能」  [実行開始] │  ← 編集・作成中は非表示にしたい
├─────────────────────────────────────────┤
│ 概要 | 実行履歴 | レビュー | 変更履歴 | 設定 │  ← 編集・作成中は非表示にしたい
├─────────────────────────────────────────┤
│ フォーム                                  │
└─────────────────────────────────────────┘
```

### 目標の画面構成（編集・作成モード時）
```
┌─────────────────────────────────────────┐
│ フォーム                                  │
└─────────────────────────────────────────┘
```

## 変更対象ファイル

### `apps/web/src/pages/TestSuiteCases.tsx`

以下の3つのケースでヘッダーを非表示にする:

1. **テストスイート編集モード** (`isEditMode = true`): 242-278行目
2. **テストケース作成モード** (`isCreateMode = true`): 306-316行目
3. **テストケース編集モード** (`isTestCaseEditMode = true`): 320-338行目

## 実装内容

### TestSuiteCases.tsx の変更

```tsx
// テストスイート編集モード時: ヘッダーを非表示
if (isEditMode) {
  return (
    <div className="h-full flex flex-col">
      {/* TestSuiteHeaderを削除 */}
      <div className="flex-1 overflow-hidden p-4">
        <div className="card h-full overflow-hidden">
          <TestSuiteForm ... />
        </div>
      </div>
    </div>
  );
}

// 通常表示時: 作成モード・テストケース編集モードでもヘッダー非表示
return (
  <div className="h-full flex flex-col">
    {/* 作成モード・テストケース編集モードでなければヘッダーを表示 */}
    {!isCreateMode && !isTestCaseEditMode && (
      <TestSuiteHeader ... />
    )}
    ...
  </div>
);
```

## 検証方法
1. テストスイート詳細画面で「編集」ボタンをクリック → ヘッダーが非表示になることを確認
2. テストケース選択時に「編集」ボタンをクリック → ヘッダーが非表示になることを確認
3. テストケース「作成」ボタンをクリック → ヘッダーが非表示になることを確認
4. 各フォームで「キャンセル」をクリック → ヘッダーが再表示されることを確認
5. 保存完了後 → ヘッダーが再表示されることを確認
