## 実装計画: Agentest 開発環境構築

### 開発方針: 完全 Docker 開発

本プロジェクトでは **ホストPC に Node.js / pnpm をインストールしない** 方式を採用します。

| 項目 | 説明 |
|------|------|
| **ホストに必要なもの** | Docker, Docker Compose のみ |
| **開発サーバー** | Docker コンテナ内で実行 |
| **依存関係管理** | コンテナ内の pnpm |
| **ファイル編集** | ホストの IDE（ボリュームマウント経由） |

**メリット:**
- 環境差異が発生しない（チーム全員が同一環境）
- セットアップが `docker compose up` だけで完了
- Node.js バージョン管理が不要

```typescript
    timestamp: new Date().toISOString(),
    uptime: process.uptime(),
```

### 5.2 Dockerfile.web (詳細版)

```dockerfile
# Stage 1: Dependencies
FROM node:22-alpine AS deps
RUN corepack enable && corepack prepare pnpm@latest --activate
WORKDIR /app
COPY pnpm-lock.yaml pnpm-workspace.yaml ./
COPY packages/shared/package.json ./packages/shared/
COPY apps/web/package.json ./apps/web/
RUN pnpm fetch

# Stage 2: Build
FROM node:22-alpine AS builder
RUN corepack enable && corepack prepare pnpm@latest --activate
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN pnpm install --offline
RUN pnpm --filter @agentest/web build

# Stage 3: Development
FROM node:22-alpine AS development
RUN corepack enable && corepack prepare pnpm@latest --activate
WORKDIR /app
COPY . .
RUN pnpm install
EXPOSE 3000
CMD ["pnpm", "--filter", "@agentest/web", "dev", "--host"]

# Stage 4: Production (nginx)
FROM nginx:alpine AS production
COPY --from=builder /app/apps/web/dist /usr/share/nginx/html
COPY docker/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
HEALTHCHECK --interval=30s --timeout=3s CMD wget -qO- http://localhost/ || exit 1
```

```bash
# docker ディレクトリに移動
cd docker

docker compose up --build

docker compose up -d --build
```

### 6.2 pnpm コマンドの実行方法

ホストに pnpm がないため、コンテナ経由でコマンドを実行します。

```bash
# 方法1: dev コンテナを使用（推奨）
docker compose --profile tools up -d dev
docker compose exec dev pnpm install
docker compose exec dev pnpm --filter @agentest/db prisma generate
docker compose exec dev pnpm --filter @agentest/db prisma migrate dev

# 方法2: ワンショットで実行
docker compose run --rm dev sh -c "corepack enable && pnpm install"

# 方法3: api コンテナ内で実行
docker compose exec api pnpm --filter @agentest/db prisma migrate dev
```

### 6.3 よく使うコマンド

```bash
# ログ確認
docker compose logs -f api
docker compose logs -f web

# コンテナ再起動
docker compose restart api

# 全停止
docker compose down

# ボリューム含めて削除（クリーンアップ）
docker compose down -v

# 特定サービスのみ起動
docker compose up -d db redis minio
```

```
- db (Prisma スキーマ - 既存設計 + 認証テーブル)
- auth (OAuth, JWT, ミドルウェア)
- storage (S3互換クライアント)
- ws-types (WebSocket イベント型)
- ui (空の雛形)
```

### Step 3: API 作成
```
apps/api を作成してください：
- Express アプリケーション
- 認証ルート (/auth/github, /auth/google, /auth/me, etc.)
- ヘルスチェック
- エラーハンドリング
```

### Step 4: WebSocket 作成
```
apps/ws を作成してください：
- ws ベースの WebSocket サーバー
- Redis Pub/Sub 連携
- JWT 認証
```

### Step 5: フロントエンド作成
```
apps/web を作成してください：
- Vite + React + TypeScript
- Tailwind CSS (ダークモード)
- ログインページ
- 認証フック
```

### Step 6: Docker 環境
```
docker/ 配下に Docker 環境を構築してください：
- docker-compose.yml (db, redis, minio, api, ws, web, admin)
- 各サービスの Dockerfile
```

### GitHub Actions ワークフロー

**.github/workflows/ci.yml:**
```yaml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 10
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint

  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: test
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
      redis:
        image: redis:7-alpine
        options: >-
          --health-cmd "redis-cli ping"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

          - 6379:6379
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 10
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile

      - run: pnpm test
        env:

          REDIS_URL: redis://localhost:6379

  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 10
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm build
```
