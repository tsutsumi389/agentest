# テストスイート画面 UI/UX改善計画

## 目標
テストスイートヘッダーのレイアウトを以下の形式に変更する：

```
[< プロジェクト名] テストスイート名              [編集] [実行開始]
[概要] [実行履歴] [レビュー] [変更履歴] [設定]
```

## 現在のレイアウト
```
プロジェクト名 > テストスイート名
[概要] [実行履歴] [レビュー] [変更履歴] [設定]     [編集] [+ テストケース] [実行開始]
```

## 変更内容

### 1. ヘッダー構造の変更

**変更前:**
- 1行目: パンくずリスト（プロジェクト名 > テストスイート名）
- 2行目: タブ + アクションボタン（右寄せ）

**変更後:**
- 1行目: 戻るボタン（< プロジェクト名） + テストスイート名（ページタイトル） + アクションボタン（右寄せ）
- 2行目: タブのみ

### 2. 具体的なUI変更

#### 戻るボタン
- `< プロジェクト名` 形式（ChevronLeftアイコン + プロジェクト名）
- クリックでプロジェクト詳細画面へ遷移
- テキストカラー: `text-foreground-muted`, ホバー時: `text-foreground`

#### テストスイート名
- パンくずから独立させ、ページタイトルとして表示
- `text-lg font-semibold text-foreground`

#### アクションボタン
- 1行目の右側に配置
- 「+ テストケース」ボタンは削除（サイドバーの+ボタンで代用）
- 残るボタン: 「編集」「実行開始」

#### タブ
- 現状維持（2行目に単独配置）

### 3. テストケース選択時
- 現状維持（パンくずリスト形式）

## 対象ファイル

| ファイル | 変更内容 |
|---------|----------|
| `apps/web/src/components/test-suite/TestSuiteHeader.tsx` | ヘッダーレイアウト変更 |

## 実装手順

1. テストスイート表示時（!isTestCaseMode）のパンくずリスト部分を変更
   - パンくずリスト → 戻るボタン + テストスイート名 + アクションボタン
2. アクションボタンを1行目に移動
3. 「+ テストケース」ボタンを削除
4. タブ行からアクションボタンを削除（タブのみ残す）

## 検証方法

1. `cd docker && docker compose up` でフロントエンド起動
2. テストスイート画面にアクセス
3. 確認項目:
   - 戻るボタン（< プロジェクト名）の表示と動作
   - テストスイート名のページタイトル表示
   - 「編集」「実行開始」ボタンの動作
   - タブの動作
   - テストケース選択時は従来どおりのパンくずリスト表示
