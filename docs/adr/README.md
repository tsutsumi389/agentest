# Architecture Decision Records (ADR)

技術的な意思決定を記録するドキュメントです。

## ADR とは

- **なぜその技術を選んだか**を記録
- 将来の自分やチームメンバーへの説明
- 意思決定の経緯を追跡可能に

## フォーマット

```markdown
# ADR-XXXX: タイトル

## ステータス
提案 / 採用 / 非推奨 / 却下

## コンテキスト
どのような問題や状況があったか

## 決定
何を決定したか

## 結果
この決定によるメリット・デメリット
```

1. 次の番号でファイル作成: `XXXX-<slug>.md`
2. フォーマットに従って記述
3. この README の一覧に追加
