# セキュリティ対策

最終更新日: 2025年12月

## 1. 概要

Agentestは、ユーザーデータの保護とサービスのセキュリティを最優先事項としています。本ドキュメントでは、実施しているセキュリティ対策について説明します。

## 2. インフラストラクチャセキュリティ

### 2.1 ホスティング

| 項目 | 詳細 |
|------|------|
| プロバイダ | AWS / GCP |
| リージョン | 東京 (主) + 大阪 (DR) |
| コンプライアンス | SOC 2 Type II, ISO 27001 |

### 2.2 ネットワークセキュリティ

```
[インターネット]
      │
      ▼
┌──────────────┐
│   WAF/CDN    │  DDoS保護、ボット対策
└──────────────┘
      │
      ▼
┌──────────────┐
│ Load Balancer│  TLS終端、ヘルスチェック
└──────────────┘
      │
      ▼
┌──────────────┐
│  App Servers │  プライベートサブネット
└──────────────┘
      │
      ▼
┌──────────────┐
│   Database   │  暗号化、アクセス制限
└──────────────┘
```

### 2.3 ネットワーク制御

- **VPC分離**: 本番/ステージング/開発環境の分離
- **セキュリティグループ**: 最小権限のポート開放
- **プライベートサブネット**: データベースは外部アクセス不可

## 3. アプリケーションセキュリティ

### 3.1 認証・認可

| 対策 | 実装 |
|------|------|
| パスワード | bcrypt (cost=12) |
| セッション | JWT + Refresh Token |
| OAuth | GitHub, Google |
| MFA | TOTP対応 (Pro以上) |
| RBAC | 組織/プロジェクトレベル |

### 3.2 通信の暗号化

| 層 | プロトコル |
|----|-----------|
| 外部通信 | TLS 1.3 |
| 内部通信 | mTLS |
| WebSocket | WSS |

### 3.3 入力検証

```typescript
// Zodによるバリデーション例
const TestCaseSchema = z.object({
  title: z.string().min(1).max(200),
  description: z.string().max(5000).optional(),
  priority: z.enum(['low', 'medium', 'high', 'critical']),
});
```

### 3.4 セキュリティヘッダー

```typescript
// Helmet設定
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'"],
    }
  },
  hsts: { maxAge: 31536000, includeSubDomains: true },
  referrerPolicy: { policy: 'strict-origin-when-cross-origin' },
}));
```

## 4. データセキュリティ

### 4.1 データ暗号化

| データ状態 | 暗号化方式 |
|-----------|-----------|
| 保存時 | AES-256-GCM |
| 転送時 | TLS 1.3 |
| バックアップ | AES-256 + KMS |

### 4.2 機密データの取り扱い

| データ種別 | 保護措置 |
|-----------|---------|
| パスワード | ハッシュ化 (bcrypt) |
| APIトークン | ハッシュ保存、プレフィックス表示 |
| 支払い情報 | 決済代行会社で管理 |
| 個人情報 | 暗号化 + アクセスログ |

### 4.3 データ分離

- **組織間分離**: 論理的に完全分離
- **テナント識別**: すべてのクエリに組織IDフィルタ
- **監査ログ**: データアクセスの全記録

## 5. 脆弱性対策

### 5.1 OWASP Top 10 対策

| 脆弱性 | 対策 |
|--------|------|
| インジェクション | パラメータ化クエリ (Prisma) |
| 認証の不備 | セキュアなセッション管理 |
| 機密データの露出 | 暗号化、マスキング |
| XXE | XMLパーサー無効化 |
| アクセス制御の不備 | RBAC、オブジェクトレベル認可 |
| セキュリティ設定ミス | セキュリティヘッダー、最小権限 |
| XSS | CSP、出力エスケープ |
| 安全でないデシリアライゼーション | JSONのみ使用 |
| 既知の脆弱性 | 依存関係の定期更新 |
| 不十分なログ | 監査ログ、アラート |

### 5.2 セキュリティテスト

| テスト種別 | 頻度 |
|-----------|------|
| 静的解析 (SAST) | CI/CD毎 |
| 依存関係スキャン | 毎日 |
| 動的解析 (DAST) | 週次 |
| ペネトレーションテスト | 年次 |

### 5.3 依存関係管理

```bash
# 脆弱性チェック (CI/CD)
pnpm audit --audit-level=high

# Renovateによる自動更新
# セキュリティパッチは即座にマージ
```

## 7. 監視とログ

### 7.1 ログ収集

| ログ種別 | 保持期間 | 目的 |
|---------|---------|------|
| アクセスログ | 90日 | 分析、障害調査 |
| 認証ログ | 1年 | セキュリティ監査 |
| 監査ログ | 7年 | コンプライアンス |
| エラーログ | 30日 | デバッグ |

### 7.2 アラート

| イベント | 対応 |
|---------|------|
| 認証失敗(閾値超え) | 即時通知、一時ブロック |
| 異常なAPI使用 | 調査開始 |
| 権限昇格試行 | 即時通知、調査 |
| データ大量エクスポート | 通知、確認 |

### 7.3 SIEM連携

Enterpriseプランでは、SIEMツール (Splunk, Datadog等) への ログ連携が可能です。
