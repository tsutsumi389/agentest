# サービスレベルアグリーメント (SLA)

最終更新日: 2025年12月

## 1. 概要

本SLA（Service Level Agreement）は、Agentestのサービス品質に関するコミットメントを定義します。本SLAはTeamプラン以上のユーザーに適用されます。

## 2. サービス可用性

### 2.1 可用性目標

| プラン | 月間稼働率目標 | 許容ダウンタイム/月 |
|--------|---------------|-------------------|
| Free | - | SLA対象外 |
| Pro | 99.5% | 約3.6時間 |
| Team | 99.9% | 約43分 |
| Enterprise | 99.95% | 約22分 |

### 2.2 可用性の計算

```
月間稼働率 = (総時間 - ダウンタイム) / 総時間 × 100
```

**除外されるダウンタイム:**
- 計画メンテナンス（事前通知あり）
- ユーザー起因の問題
- 不可抗力（天災等）
- サードパーティサービスの障害

## 3. サービスクレジット

SLAを満たせなかった場合、以下のサービスクレジットを提供します。

| 月間稼働率 | クレジット |
|-----------|-----------|
| 99.0% - 99.9% | 月額料金の10% |
| 95.0% - 99.0% | 月額料金の25% |
| 95.0%未満 | 月額料金の50% |

### クレジット申請方法

1. 障害発生から30日以内にサポートへ申請
2. 対象期間とダウンタイムを明記
3. 審査後、翌月の請求から控除

## 4. パフォーマンス目標

### 4.1 レスポンスタイム

| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| API p50 | 100ms以下 | サーバー処理時間 |
| API p95 | 500ms以下 | サーバー処理時間 |
| API p99 | 1000ms以下 | サーバー処理時間 |
| ページ読み込み | 3秒以下 | フルロード |

### 4.2 リアルタイム同期

| 指標 | 目標値 |
|------|--------|
| 更新反映 | 1秒以内 |
| WebSocket遅延 | 100ms以下 |
| 同時編集競合解決 | 500ms以内 |

## 5. データ保護

### 5.1 バックアップ

| 項目 | 内容 |
|------|------|
| バックアップ頻度 | 毎日（増分）、毎週（完全） |
| バックアップ保持期間 | 30日間 |
| リストア時間目標 | 4時間以内 |
| 地理的冗長化 | 2リージョン以上 |

### 5.2 災害復旧

| 指標 | Team | Enterprise |
|------|------|------------|
| RPO (目標復旧時点) | 1時間 | 15分 |
| RTO (目標復旧時間) | 4時間 | 1時間 |

## 6. サポートレスポンス

### 6.1 サポートレベル

| プラン | サポート種別 | 対応時間 |
|--------|-------------|---------|
| Free | コミュニティ | - |
| Pro | メール | 営業日 |
| Team | メール + チャット | 営業日 |
| Enterprise | 専用サポート | 24/365 |

### 6.2 初回応答時間

| 優先度 | 定義 | Pro | Team | Enterprise |
|--------|------|-----|------|------------|
| Critical | サービス全体停止 | - | 1時間 | 15分 |
| High | 主要機能の障害 | 24時間 | 4時間 | 1時間 |
| Medium | 一部機能の問題 | 48時間 | 8時間 | 4時間 |
| Low | 質問・要望 | 72時間 | 24時間 | 8時間 |

## 7. 計画メンテナンス

### 7.1 メンテナンスウィンドウ

- **定期メンテナンス**: 毎月第3日曜日 AM2:00-6:00 (JST)
- **緊急メンテナンス**: セキュリティ上必要な場合、随時

### 7.2 通知タイミング

| メンテナンス種別 | 事前通知 |
|-----------------|---------|
| 定期（影響なし） | 3日前 |
| 定期（影響あり） | 7日前 |
| 緊急 | 可能な限り早く |

### 7.3 通知方法

- ステータスページ (status.agentest.io)
- 管理者メール
- サービス内バナー

## 8. セキュリティ

### 8.1 セキュリティ対応

| 項目 | コミットメント |
|------|---------------|
| 脆弱性スキャン | 週次 |
| ペネトレーションテスト | 年次 |
| セキュリティインシデント対応 | 24時間以内に初期対応 |

詳細は[セキュリティ対策](./security.md)を参照してください。

## 9. ステータスページ

サービスの稼働状況は以下で確認できます:

- **ステータスページ**: https://status.agentest.io
- **稼働履歴**: 過去90日間の稼働状況
- **インシデント履歴**: 過去の障害と対応

### ステータス表示

| ステータス | 説明 |
|-----------|------|
| Operational | 正常稼働 |
| Degraded Performance | パフォーマンス低下 |
| Partial Outage | 一部障害 |
| Major Outage | 重大障害 |
| Maintenance | メンテナンス中 |

## 10. SLAの変更

- SLAの変更は30日前までに通知します。
- 変更がサービスレベルの低下を伴う場合、影響を受けるユーザーは契約を解除できます。
